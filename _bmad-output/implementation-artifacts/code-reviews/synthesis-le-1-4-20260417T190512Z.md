- **Claimed Issue:** "AC #3 missing — POST size rejection not immediate, only at end of body" | **Raised by:** Validator B | **Dismissal Reason:** Factually incorrect. The ESPAsyncWebServer body callback receives `total` (the declared Content-Length) on the **first invocation** (`index == 0`). The guard at WebPortal.cpp line 803 (`if (total > LAYOUT_STORE_MAX_BYTES)`) runs before the `if (index == 0)` accumulation block — it fires on the first chunk and returns 413 immediately via `clearPendingBody() + _sendJsonError()` before any buffering occurs. The validator confused "end of body" with "first callback" — the `total` field is the full declared size from the HTTP request headers, not the accumulated bytes so far.

AC #8 says "no partial file is left on LittleFS (LayoutStore::save already guarantees this — WebPortal just maps the error code)" — the test responsibility is on LayoutStore, not WebPortal. ### Validator C MINOR Issues

- **Claimed Issue:** "Body handler race condition — `onDisconnect` could call `clearPendingBody` mid-accumulation" | **Raised by:** Validator C | **Dismissal Reason:** This is the existing project-wide pattern used by `POST /api/settings`, `POST /api/display/mode`, and `POST /api/schedule`. The disconnect handler fires on the TCP/IP task, but ESPAsyncWebServer serializes disconnect handling through the async queue — the body callback and disconnect callback don't interleave in the mathieucarbou fork. This concern is real in theory but is architectural to all JSON POST endpoints in the firmware, not introduced by LE-1.4. Out of scope for LE-1.4.

- **Claimed Issue:** "Static JSON builder heap usage — pre-serialize widget schema to avoid runtime JSON construction" | **Raised by:** Validator C | **Dismissal Reason:** The `_handleGetWidgetTypes` response is ~1.2 KB serialized. On a 4 MB flash device with this endpoint called rarely, the tradeoff is not worthwhile. The v7 JsonDocument is heap-efficient and destructed on handler return. Noted as a possible micro-optimization but not a defect.
