## 🏛️ Architectural Sins

- **[6/10] Dependency Inversion:** `WebPortal.cpp` tightly couples OTA upload logic to `ModeRegistry::prepareForOTA()`, hardcoding a core orchestration dependency into a web adapter.
  - 📍 `firmware/adapters/WebPortal.cpp:656`
  - 💡 Fix: Introduce an `OTAOrchestrator` interface or use an event-based system to decouple the web adapter from the core state management.

- **[🔴 9/10] Boundary breach:** `ModeRegistry::prepareForOTA()` mutates `_activeMode` directly on Core 1 (`firmware/core/ModeRegistry.cpp:478-482`) even though `ModeRegistry::tick()` and `_activeMode->render()` run on Core 0. `SwitchState::SWITCHING` is not consulted before rendering, and `_otaMode` is only a plain `bool`. This is exactly the cross-core ownership violation documented in the architecture.
  - 📍 `firmware/core/ModeRegistry.cpp:478`

## 🐛 Correctness &amp; Safety

- **🐛 Push OTA failure path never restores the renderer:** After `ModeRegistry::prepareForOTA()` succeeds in the upload handler (`firmware/adapters/WebPortal.cpp:656-675`), later failures only invalidate the request and abort the update, but never call `ModeRegistry::completeOTAAttempt(false)`. The `main.cpp` display task keeps taking the OTA branch forever, soft-sticking the wall on "Updating..." until manual reboot.
  - 📍 `firmware/adapters/WebPortal.cpp:692`
  - 🔄 Reproduction: Simulate an HTTP upload failure (e.g., bad image magic byte) via `POST /api/ota/upload`.

- **🔒 [High] Insecure TLS usage:** `OTAUpdater.cpp` uses `binClient.setInsecure()` for GitHub firmware downloads. While noted as an architectural tradeoff, it exposes the firmware update process to Man-in-the-Middle (MITM) attacks.
  - 📍 `firmware/core/OTAUpdater.cpp:455`
  - ⚠️ Impact: Attackers can replace the firmware binary with malicious code.

- **🎭 Lying Test:** `test_start_download_rejects_when_not_available`
  - 📍 `firmware/test/test_ota_updater/test_main.cpp:259`
  - 🤥 Why it lies: The test relies on the internal `_resetForTest()` helper, which is not available in production code. It does not test the actual state machine interaction with the network stack or orchestrator.
