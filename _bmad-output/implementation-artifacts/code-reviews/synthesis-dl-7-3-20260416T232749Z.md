**Finding #2: setInterval Polling Pattern Creates Overlapping HTTP Requests**
- **Source**: Validator A, Validator B (consensus)
- **File**: `firmware/data-src/dashboard.js` (lines 3833-3836, 3839-3905)
- **Evidence**: Original code used `setInterval(pollOtaStatus, 500)` where `pollOtaStatus` contains async `FW.get('/api/ota/status')`. Under network latency or device load, new polls trigger before previous fetch completes, creating request stacking on resource-constrained ESP32.
- **Fix**: **APPLIED** - Replaced `setInterval` with recursive `setTimeout` pattern. Each poll now schedules next iteration only after current fetch completes. Updated all `clearInterval` calls to `clearTimeout`.
- **Impact**: Prevents request stacking, reduces peak concurrent load on ESP32 web server, safer lifecycle for backgrounded browser tabs

## Changes Applied

**File**: `firmware/data-src/dashboard.js`
**Change**: Replaced setInterval polling with recursive setTimeout pattern

**Before**:
```javascript
function startOtaPollLoop() {
  if (otaPollTimer) clearInterval(otaPollTimer);
  otaPollTimer = setInterval(pollOtaStatus, OTA_POLL_INTERVAL);
}
```

**After**:
```javascript
function startOtaPollLoop() {
  if (otaPollTimer) clearTimeout(otaPollTimer);
  otaPollTimer = null;
  pollOtaStatus(); // Start first poll immediately
}

function pollOtaStatus() {
  FW.get('/api/ota/status').then(function(res) {
    // ... [state handling] ...
    if (d.state === 'downloading') {
      // ... [progress UI] ...
      otaPollTimer = setTimeout(pollOtaStatus, OTA_POLL_INTERVAL);
    } else if (d.state === 'verifying') {
      // ... [verification UI] ...
      otaPollTimer = setTimeout(pollOtaStatus, OTA_POLL_INTERVAL);
    }
    // ... [other states] ...
  }).catch(function() {
    // ... [error handling] ...
  });
}
```

**Rationale**: Prevents HTTP request stacking under network latency by ensuring next poll only schedules after previous fetch completes.
