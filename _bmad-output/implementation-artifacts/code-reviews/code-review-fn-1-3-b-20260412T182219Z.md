## ⚡ Performance & Scalability

- **[Medium] Dynamic String Allocations in Structs:** The `OTAUploadState` struct uses `String` for `error` and `errorCode` members. Although ESPAsyncWebServer chunks are handled efficiently, dynamic string allocations and deallocations can contribute to heap fragmentation on a resource-constrained ESP32, especially if errors are frequent. This contradicts the spirit of the "no new/delete" rule in architecture.
  - 📍 `firmware/adapters/WebPortal.cpp:83`
  - 💡 Fix: Consider using fixed-size char arrays (e.g., `char error[64]`) or `const char*` literals for error messages to avoid dynamic memory allocation for error strings, or implement a custom string class that uses a pre-allocated buffer.
