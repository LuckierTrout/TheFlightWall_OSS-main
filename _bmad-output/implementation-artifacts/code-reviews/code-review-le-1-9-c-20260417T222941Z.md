## Architectural Sins

- **[6/10] Interface honesty / RenderContext contract abuse (test vs prod):** Production `CustomLayoutMode::render` requires a live `FastLED_NeoMatrix*`; tests systematically use `nullptr` to mean “hardware-free,” which is a different contract from “render custom layout.” The story’s AC #1 language (“render through `CustomLayoutMode::render()` … matches golden”) is not satisfied in spirit—only in a redefined, weaker sense (`TEST_PASS` after a no-op return).
  - `firmware/modes/CustomLayoutMode.cpp` (`render` early return on null matrix)
  - **Fix:** Add a compile-time test-only stub matrix type, or a `PIO_UNIT_TESTING` branch that uses a minimal framebuffer-backed matrix for `CustomLayoutMode::render` only in tests (still on-device), plus deterministic clock injection.

- **Abstraction leak / gate semantics:** Size gate logic lives in SCons Python with special-case `PROJECT_DIR` handling; fine, but the *policy* (merge-base vs committed int) is where the architecture of release gating diverges from the story spec.

---

## Pythonic Crimes & Readability

- **Broad exception swallowing** in `get_main_baseline_size` (`except Exception: return None`) — prefer catching `ValueError`/`OSError` explicitly and logging a one-line reason before skipping delta gate.
- **Magic numbers** duplicated (`0x180000`, `0.92`, `180 * 1024`) — acceptable for a small script, but the verification report should not re-derive them incorrectly.

## Performance & Scalability

- **Init-time logging on hot path for layout reloads:** `Serial.printf` heap prints on every `CustomLayoutMode::init` success/failure add serial I/O latency during layout activation bursts (editor UX). Prefer `LOG_*` macros with levels or guard behind `#ifdef DEBUG_LAYOUT`.

## Lying Tests (explicit)

1. **`test_golden_*_render_does_not_crash` for A/B/C:** Calls `CustomLayoutMode::render` with `ctx.matrix == nullptr`, which returns immediately — the test proves almost nothing about rendering.
2. **`test_golden_b_flight_field_resolves_airline` / `test_golden_b_metric_resolves_alt`:** Useful as widget unit smoke, but mislabeled as golden/integration for fixture B through `CustomLayoutMode`.
3. **Heap floor test (`test_golden_c_heap_floor`):** Still valuable on-device, but it does not measure heap impact of the real render loop (widgets never draw because matrix is null), so it under-samples the worst-case path described in AC #4’s intent (“during golden-frame test run” while layout is active).
