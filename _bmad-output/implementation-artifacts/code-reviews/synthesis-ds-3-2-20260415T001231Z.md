## Issues Verified (by severity)

### Critical

### High

- **Issue**: Race condition — `WebPortal.cpp` (Core 1) calls `ModeRegistry::getLastError()` which returns a raw pointer to `_lastError[]`, a 64-byte static buffer written by Core 0 via `snprintf()`. JSON serialization of this pointer races against the Core 0 write. `copyLastError()` was declared in `ModeRegistry.h` as the safe alternative but was **never implemented** in the `.cpp`. | **Source**: Validator A | **File**: `firmware/adapters/WebPortal.cpp:1092,1227`, `firmware/core/ModeRegistry.cpp` | **Fix**: Implemented `copyLastError()` with seqlock-style double `_errorUpdating` check in `ModeRegistry.cpp`; replaced both `getLastError()` call sites in `WebPortal.cpp` with `copyLastError()` into stack-allocated `char[64]` buffers.

- **Issue**: Memory leak on mode init failure — `ModeRegistry::executeSwitch()` calls `delete newMode` without calling `newMode->teardown()` first. Architecture rule D2 explicitly requires teardown-before-delete. If `init()` partially allocates heap before returning false, those resources are permanently leaked. | **Source**: Validator A | **File**: `firmware/core/ModeRegistry.cpp:241` | **Fix**: Added `newMode->teardown()` immediately before `delete newMode` in the init-failure path.

### Medium

- **Issue**: Tautological test — `test_first_switch_fail_queues_recovery_activation` seeds NVS with `"mock_mode_a"`, triggers a failed switch of `mock_mode_a`, then asserts NVS reads back `"mock_mode_a"`. If recovery logic never ran at all and NVS was untouched, the test still passes. The system under test cannot be falsified. | **Source**: Validator A | **File**: `firmware/test/test_mode_registry/test_main.cpp:525` | **Fix**: Changed seed value from `"mock_mode_a"` to `"stale_invalid_mode"`. Now, if recovery logic fails to overwrite NVS, the assertion fails correctly.

- **Issue**: String `reserve()` capacity lost on `push_back` — all three POST body accumulation sites construct a `PendingRequestBody` with `body.reserve(total)`, then `push_back` the struct by value. The Arduino `String` copy constructor allocates exactly the current content length (0), discarding the reserved capacity. Subsequent `concat()` calls cause repeated heap reallocations on a fragmented ESP32 heap. | **Source**: Validator A | **File**: `firmware/adapters/WebPortal.cpp:215-217, 332-334, 377-379` | **Fix**: Changed pattern to `push_back({request, String()})` first, then `g_pendingBodies.back().body.reserve(total)` on the stored element. Applied to all 3 handlers (settings, display/mode, schedule).

- **Claimed Issue**: Rigid JSON validation (`is<int32_t>()`) rejects boolean values in mode settings, violating OCP since the schema type field supports `"bool"`. | **Raised by**: Validator A | **Dismissal Reason**: No current schema uses `"bool"` type — both deployed schemas use `"enum"` and `"uint8"`. Deferred as a future-proofing improvement when bool-typed settings are added (see Suggested Future Improvements).

- **Claimed Issue**: Partial-apply risk — `ConfigManager::setModeSetting()` has secondary validation; if it returns false mid-loop, NVS is partially applied. | **Raised by**: Validator A | **Dismissal Reason**: Both current schemas have exactly 1 setting each (clock: `format`; depbd: `rows`). Partial apply requires ≥2 settings. Additionally, the secondary validation ranges in `setModeSetting()` are identical to the schema's `minValue`/`maxValue` already checked in pre-validation. The only other failure path (`prefs.begin()` NVS hardware failure) would fail on the first write and is not a "partial apply" scenario. Not a current correctness issue.

**Change**: Implemented missing `copyLastError()` — thread-safe snapshot with seqlock-style double-check of `_errorUpdating`
**Before**:
```cpp

```
**After**:
```cpp
void ModeRegistry::copyLastError(char* outBuf, size_t maxLen) {
    if (outBuf == nullptr || maxLen == 0) return;
    outBuf[0] = '\0';
    if (_errorUpdating.load()) return;
    strlcpy(outBuf, _lastError, maxLen);
    if (_errorUpdating.load()) outBuf[0] = '\0';
}
bool ModeRegistry::prepareForOTA() {
```
