- **Claimed Issue**: "Missing `g_flightCount` atomic (architecture.md#DL2)" | **Raised by**: Reviewer A | **Dismissal Reason**: The `g_flightCount` atomic in architecture.md is specified for Core 1 → Core 0 signaling when ModeRegistry::tick() runs in the display task (Core 0). Story dev notes explicitly address this: *"ModeOrchestrator state is read/written from Core 1 only (orchestrator tick + web handler), so no atomic needed."*
