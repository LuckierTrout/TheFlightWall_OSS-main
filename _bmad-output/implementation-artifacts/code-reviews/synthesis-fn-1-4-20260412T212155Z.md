## Issues Dismissed

- **Claimed**: AC #6 violated — rollback WARNING emitted on normal boots after historical rollback | **Raised by**: Reviewer A | **Dismissal**: Already dismissed in round 1. Story Gotcha #2 explicitly designates this as intentional: `g_rollbackDetected` persists across boots until next successful OTA, and the WARNING is the mechanism that surfaces this state via the health API. Suppressing it would hide a real device condition.

- **Claimed**: String concatenation overhead — `String okMsg = "..." + String(elapsedSec) + "s"` causes heap fragmentation | **Raised by**: Reviewer B | **Dismissal**: Already fixed in round 1 — code uses named `String` variables (the prior anti-pattern was `String(...).c_str()` on a temporary). The `snprintf` alternative is a style preference with negligible benefit at one-shot call frequency.

- **Claimed**: Redundant `if (g_otaSelfCheckDone) return;` inside function — loop() already guards with `if (!g_otaSelfCheckDone)` | **Raised by**: Reviewer B | **Dismissal**: Defense-in-depth guard. If `performOtaSelfCheck()` is ever called from another location in future, the guard prevents double-execution. Not redundant in a maintenance sense.

- **Claimed**: FW_VERSION fallback `#ifndef` duplicated in main.cpp and SystemStatus.cpp | **Raised by**: Reviewer B | **Dismissal**: Minor DRY violation. Story Dev Notes explicitly instruct adding the guard to `SystemStatus.cpp` for unit test harness compatibility. Both locations serve distinct compilation contexts. The shared-header refactor is a future cleanup task, not a defect.

- **Claimed**: SystemStatus "best-effort" mutex fallback is unsafe — race condition risk | **Raised by**: Reviewer B | **Dismissal**: Pre-existing pattern across entire `SystemStatus` implementation, already dismissed in round 1. Requires broader architectural refactor outside story scope.

- **Claimed**: Magic numbers `2000/4000/2100` in `tickStartupProgress()` | **Raised by**: Reviewer B | **Dismissal**: Pre-existing code from a prior story, not introduced by fn-1.4. Out of scope.

- **Claimed**: Magic numbers `0x180000/0xF0000` in `validatePartitionLayout()` | **Raised by**: Reviewer B | **Dismissal**: Pre-existing code from fn-1.1. Out of scope. Cross-reference comment is already present.

- **Claimed**: LOG macros inconsistent with `Serial.printf` | **Raised by**: Reviewer B | **Dismissal**: Already dismissed in round 1. `LOG_E/W/I/V` macros accept only fixed string literals; `Serial.printf` is required when embedding runtime values. Mixed use is the correct project pattern.

- **Claimed**: SRP violation — `performOtaSelfCheck()` mixes state management, timing, WiFi check, OTA API, logging, SystemStatus | **Raised by**: Reviewer B | **Dismissal**: Acceptable complexity for embedded firmware. Extracting to an OtaSelfCheckManager class adds abstraction overhead without proportionate testability benefit at this project scale.

- **Claimed**: Race condition — `g_wifiManager.getState()` accessed from loop() without synchronization | **Raised by**: Reviewer C | **Dismissal**: False positive. Story Dev Notes explicitly state "Self-check runs on Core 1 — loop() runs on Core 1, which is correct. WiFiManager's WiFi stack also runs on Core 1. No cross-core issues." The call is safe by architecture.

- **Claimed**: millis() overflow at 49.7 days | **Raised by**: Reviewer C | **Dismissal**: Not a real bug. The self-check completes at 60 seconds max. Unsigned subtraction (`millis() - g_bootStartMs`) correctly handles wraparound on all platforms including ESP32. Even a wraparound would produce a very large `elapsed` value, correctly triggering the 60s timeout.

- **Claimed**: Lying test `test_system_status_ota_no_change_on_normal_boot()` — doesn't call `performOtaSelfCheck()` so can't verify the "no set" contract | **Raised by**: Reviewer C | **Dismissal**: The test comment explicitly acknowledges the limitation. The test does verify the correct initial state contract, which is meaningful. Renaming is cosmetic. The underlying limitation (static function not directly testable) is captured as an action item.

- **Claimed**: Unnecessary loop overhead after `g_otaSelfCheckDone = true` | **Raised by**: Reviewer C | **Dismissal**: A single boolean check per loop iteration is negligible overhead (~1ns). The pattern is idiomatic for one-shot initialization in Arduino loop().

- **Claimed**: No automated test for retry-on-error behavior | **Raised by**: Reviewer C | **Dismissal**: Valid test gap, but not a code defect. Captured as existing action item.

- **Claimed**: FW_VERSION format not validated at runtime | **Raised by**: Reviewer B | **Dismissal**: `FW_VERSION` is a compile-time build flag enforced by `platformio.ini` and team convention. Runtime validation of a compile-time constant is over-engineering with no failure mode that warrants it.

## Changes Applied

**File**: `firmware/src/main.cpp`
**Change 1**: Fix `s_isPendingVerify` error caching — removed premature default assignment
```cpp
// BEFORE:
s_isPendingVerify = 0;
if (running && esp_ota_get_state_partition(running, &state) == ESP_OK) {
    s_isPendingVerify = (state == ESP_OTA_IMG_PENDING_VERIFY) ? 1 : 0;
}

// AFTER:
if (running && esp_ota_get_state_partition(running, &state) == ESP_OK) {
    s_isPendingVerify = (state == ESP_OTA_IMG_PENDING_VERIFY) ? 1 : 0;
}
// If running is NULL or state probe fails, s_isPendingVerify stays -1 and retries next call.
```

**File**: `firmware/src/main.cpp`
**Change 2**: Move rollback WARNING before mark_valid call + add log spam throttle
```cpp
// BEFORE (rollback inside err==ESP_OK block, log spam on failure):
if (err == ESP_OK) {
    ...
    if (g_rollbackDetected) { SystemStatus::set(WARNING, "rolled back"); }
    g_otaSelfCheckDone = true;
} else {
    Serial.printf("[OTA] ERROR: ...\n");  // fires every loop iteration
    SystemStatus::set(ERROR, errMsg);
}

// AFTER (rollback before mark_valid; throttled error log):
if (g_rollbackDetected) {
    static bool s_rollbackStatusSet = false;
    if (!s_rollbackStatusSet) {
        SystemStatus::set(WARNING, "Firmware rolled back to previous version");
        s_rollbackStatusSet = true;
    }
}
esp_err_t err = esp_ota_mark_app_valid_cancel_rollback();
if (err == ESP_OK) { ... g_otaSelfCheckDone = true; }
else {
    static bool s_markValidErrorLogged = false;
    if (!s_markValidErrorLogged) {
        Serial.printf("[OTA] ERROR: ...\n"); SystemStatus::set(ERROR, errMsg);
        s_markValidErrorLogged = true;
    }
}
```

**File**: `firmware/src/main.cpp`
**Change 3**: Move boot timestamp before Serial/delay
```cpp
// BEFORE:
Serial.begin(115200);
delay(200);
g_bootStartMs = millis();  // 200ms late

// AFTER:
g_bootStartMs = millis();  // first statement in setup()
Serial.begin(115200);
delay(200);
```
