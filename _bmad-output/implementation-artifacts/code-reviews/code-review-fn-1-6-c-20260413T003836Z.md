## 🏛️ Architectural Sins

- **[7/10] Missing Abstraction for OTA State Machine:** The OTA upload logic is tightly coupled to DOM manipulation with no separation of concerns. The `uploadFirmware()` function handles XHR, DOM updates, and state management all in one. Consider an `OTAManager` class.
  - 📍 `dashboard.js:2740-2789`
  - 💡 Fix: Extract OTA logic into a state machine class with clear states (IDLE, VALIDATING, UPLOADING, REBOOTING)

- **[6/10] No Request Cancellation Pattern:** Once `xhr.send()` is called, there's no way to abort the upload if the user navigates away or changes their mind. This wastes bandwidth and flash write cycles.
  - 📍 `dashboard.js:2788`
  - 💡 Fix: Store XHR reference and expose `abortUpload()` method wired to a cancel button during upload

---

## ⚡ Performance & Scalability

- **[HIGH] Unthrottled Progress Events:** The `xhr.upload.onprogress` handler updates the DOM on every browser progress event, which can fire tens of times per second for large files. AC #6 explicitly requires "updates at least every 2 seconds" - this is a violation.
  - 📍 `dashboard.js:2749-2754`
  - 💡 Fix: Add throttling: `if (Date.now() - lastUpdate < 500) return;` or use requestAnimationFrame

- **[MEDIUM] Missing Debounce on File Selection:** Rapid file selection changes could queue multiple FileReader operations.
  - 📍 `dashboard.js:2690-2695`
  - 💡 Fix: Abort previous FileReader before starting new validation

---

## 🐛 Correctness & Safety

- **[HIGH] Double-Upload Risk:** The upload button is not disabled when `uploadFirmware()` is called. A double-click or accidental rapid tap could start two concurrent uploads, corrupting the flash.
  - 📍 `dashboard.js:2733-2738`
  - 🔄 Reproduction: Double-click "Upload Firmware" button quickly
  - 💡 Fix: Add `btnUploadFirmware.disabled = true` at start of upload

- **[MEDIUM] XHR Reference Leak:** The XHR object is created locally but never cleaned up if the upload completes successfully. While GC will eventually collect it, there's no explicit `xhr = null` or abort on page unload.
  - 📍 `dashboard.js:2741`
  - 🔄 Reproduction: Upload firmware, navigate away without waiting for reboot
  - 💡 Fix: Store XHR reference and clean up in `resetOtaUploadState()`

- **[LOW] Filename XSS (Mitigated but fragile):** While `textContent` is used (safe), the filename is also passed to `FormData.append()` which could be problematic if the server doesn't properly validate. The server validates magic byte but not filename.
  - 📍 `dashboard.js:2743`
  - 💡 Fix: Sanitize filename server-side in WebPortal.cpp before logging

  - 📍 `WebPortal.cpp:517`
