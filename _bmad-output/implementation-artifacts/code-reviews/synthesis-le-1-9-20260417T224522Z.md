## Issues Dismissed

### Reviewer C — CRITICAL: `TEST_ASSERT_GREATER_OR_EQUAL(30720u, heapAfter)` is backwards
**Claimed:** The assertion `TEST_ASSERT_GREATER_OR_EQUAL(30720u, heapAfter)` asserts `30720 >= heapAfter`, which passes when heap is critically low and fails when heap is healthy.

**Dismissal:** **FALSE POSITIVE — verified against Unity source code.** The Unity macro signature is `TEST_ASSERT_GREATER_OR_EQUAL(threshold, actual)` and its internal implementation (`UnityAssertGreaterOrLessOrEqualNumber`) fails when `actual < threshold` (line 749: `if ((actual < threshold) && (compare & UNITY_GREATER_THAN)) { failed = 1; }`). Therefore `TEST_ASSERT_GREATER_OR_EQUAL(30720u, heapAfter)` asserts `heapAfter >= 30720` — exactly the AC #4 requirement. The assertion is **correct**.

Evidence: `/Users/christianlee/App-Development/TheFlightWall_OSS-main/firmware/.pio/libdeps/esp32dev/Unity/src/unity.h` line 224 (`#define TEST_ASSERT_GREATER_OR_EQUAL(threshold, actual)`) and `unity.c` line 749.

### Reviewer C — CRITICAL (duplicate): `test_golden_c_heap_floor` heap measurement is wrong
