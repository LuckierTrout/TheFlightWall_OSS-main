## 🐛 Correctness & Safety

- **🐛 Bug:** Promise Resolution Mismatch (Dead Code and Silent Failure)
  - 📍 `firmware/data-src/dashboard.js:886`
  - 🔄 Reproduction: When `loadDisplayModes()` catches a network error, it resolves to `null` instead of rejecting the Promise. However, the three `.then(resolve, reject)` blocks in `switchMode` expect it to reject. The `reject` handlers are completely dead code. When the network fails during a poll timeout or error recovery, the code wrongly falls into the `resolve` handler, continues blindly on stale DOM, and reports incorrect recovery state.

- **🐛 Bug:** Event Bubbling Triggers Accidental Mode Swaps
  - 📍 `firmware/data-src/dashboard.js:1018`
  - 🔄 Reproduction: `showCardError()` appends `.mode-card-alert` inside the mode card. AC #4 explicitly allows clicking anywhere in the picker to dismiss the error. Because `e.stopPropagation()` is missing, clicking the alert dismisses it, but the click event bubbles directly to the parent `.mode-card`, triggering `switchMode(modeId)` again. The user trying to dismiss an error accidentally launches a new, likely doomed, mode switch attempt.
