## Issues Dismissed

- **Claimed Issue**: `extern bool isNtpSynced()` in `WebPortal.cpp` breaks hexagonal architecture | **Raised by**: Reviewer A + B | **Dismissal Reason**: FALSE POSITIVE. The file already declares `extern FlightStatsSnapshot getFlightStatsSnapshot()` at line 37 using the identical pattern. The story's Dev Notes explicitly prescribe this approach ("aligns with project rule: no new files"). This is the established project convention, not a new violation.

- **Claimed Issue**: Race condition between `tzset()` (from `configTzTime()`) and `localtime_r()` across Core 0/Core 1 | **Raised by**: Reviewer A | **Dismissal Reason**: Theoretical but low-impact. Hot-reload is a rare user-triggered action; the `tzset()` modification window is microseconds; the worst-case consequence is a single garbled time display frame during a settings write. Accepted design risk per story scope. A mutex solution would require significant infrastructure not justified by this failure mode.

- **Claimed Issue**: Missing POSIX TZ format validation in ConfigManager | **Raised by**: Reviewer A | **Dismissal Reason**: Partially valid concern, but implemented validation (non-empty string ≤40 chars) is sufficient for the story AC. A malformed TZ string passed to `configTzTime()` → `tzset()` causes silent UTC fallback — the device stays functional and the LWIP stack doesn't crash. Full POSIX TZ syntax validation is non-trivial on ESP32 newlib and is outside story scope. Accepted risk.

- **Claimed Issue**: `g_ntpSynced` never resets to `false` after initial sync (stale success on WiFi reconnect) | **Raised by**: Reviewer B | **Dismissal Reason**: Out-of-story-scope design concern. The story ACs define `ntpSynced=false` only as the pre-sync default (AC #3). LWIP auto-retries maintain clock accuracy. The flag is correctly "sticky" for the current story's contract. If cross-story NTP state invalidation is needed (e.g., for fn-2.2 scheduling), that story should define the semantics.

- **Claimed Issue**: Raw `Serial.println` violates logging contract (should use `LOG_I`) | **Raised by**: Reviewer B | **Dismissal Reason**: Acknowledged in the Dev Agent Record as a known technical constraint. `LOG_I` is a 2-argument macro (tag + string literal only); it cannot format dynamic runtime strings. The `#if LOG_LEVEL >= 2` guard follows the pre-existing pattern already in `WiFiManager.cpp:140`. Not a new violation.

- **Claimed Issue**: Unit tests are "lying" — `test_ntp_status_in_json_output` doesn't test the actual `/api/status` HTTP endpoint | **Raised by**: Reviewer A + B | **Dismissal Reason**: Valid quality observation but not a "lying test". The test name and inline comment accurately describe what it tests (`SystemStatus::toJson()` NTP fields). Full HTTP endpoint testing is not achievable in Unity/embedded unit tests without a host simulator. The story explicitly notes "unit tests cover callback logic only" (AC #1). Coverage gap is real but architectural, not deceptive.

## Changes Applied

**File**: `firmware/src/main.cpp`
**Change 1**: Add `sntp_get_sync_status()` guard in `onNtpSync()` callback

**Before**:
```cpp
static void onNtpSync(struct timeval* tv) {
    g_ntpSynced.store(true);
    SystemStatus::set(Subsystem::NTP, StatusLevel::OK, "Clock synced");
    LOG_I("NTP", "Time synchronized");
}
```
**After**:
```cpp
static void onNtpSync(struct timeval* tv) {
    if (sntp_get_sync_status() != SNTP_SYNC_STATUS_COMPLETED) {
        return;  // Ignore reset or in-progress SNTP events
    }
    g_ntpSynced.store(true);
    SystemStatus::set(Subsystem::NTP, StatusLevel::OK, "Clock synced");
    LOG_I("NTP", "Time synchronized");
}
```

**File**: `firmware/src/main.cpp`
**Change 2**: Guard `configTzTime()` with cached timezone comparison in `onChange` lambda

**Before**:
```cpp
ScheduleConfig sched = ConfigManager::getSchedule();
configTzTime(sched.timezone.c_str(), "pool.ntp.org", "time.nist.gov");
#if LOG_LEVEL >= 2
    Serial.println("[Main] Timezone hot-reloaded: " + sched.timezone);
#endif
```
**After**:
```cpp
static String s_lastAppliedTz;
ScheduleConfig sched = ConfigManager::getSchedule();
if (sched.timezone != s_lastAppliedTz) {
    s_lastAppliedTz = sched.timezone;
    configTzTime(sched.timezone.c_str(), "pool.ntp.org", "time.nist.gov");
#if LOG_LEVEL >= 2
    Serial.println("[Main] Timezone hot-reloaded: " + sched.timezone);
#endif
}
```
