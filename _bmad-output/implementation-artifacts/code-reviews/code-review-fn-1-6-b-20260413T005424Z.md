## 🐛 Correctness & Safety

### Hidden Bugs Found:

1. **Silent firmware status staleness:** The `loadFirmwareStatus()` function only runs on page load (line 2870). If the device reboots and reconnects while the page is open, the version display won't update unless user manually refreshes.
   - 📍 `dashboard.js:2639`
   - 🔄 Reproduction: 1) Load dashboard 2) Trigger OTA from another browser tab 3) Device reboots and comes back 4) Original tab shows stale version

2. **NVS write without handle verification:** In the OTA upload success handler, `otaPrefs.begin("flightwall", false)` return value is not checked before calling `putUChar()`. While the success path typically works, a full NVS partition could cause silent failure.
   - 📍 `WebPortal.cpp:618-621`
   - 🔄 Reproduction: Fill NVS partition to capacity, then perform OTA upload

3. **Missing XHR abort on page unload:** If user closes/refreshes the page during an active firmware upload, the XHR request may not be properly aborted, potentially leaving the device in a partial OTA state.
   - 📍 `dashboard.js:2791`
   - 🔄 Reproduction: Start large firmware upload, close browser tab mid-upload

## 🛠️ Suggested Fixes

### 1. Add XHR timeout and abort handling

**File:** `firmware/data-src/dashboard.js`

```javascript
// Add before xhr.open():
xhr.timeout = 120000; // 2 minute timeout
xhr.ontimeout = function() {
  if (btnUploadFirmware) btnUploadFirmware.disabled = false;
  FW.showToast('Upload timed out — try again', 'error');
  resetOtaUploadState();
};

// Add window unload handler to abort in-flight upload
window.addEventListener('beforeunload', function(e) {
  if (xhr && xhr.readyState !== XMLHttpRequest.DONE) {
    xhr.abort();
  }
});
```
