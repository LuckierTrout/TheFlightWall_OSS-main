## Issues Verified (by severity)

### Critical

**Issue 1: Pull OTA continues even when `prepareForOTA()` fails**
- **Source**: Both Reviewer A (+3) and Reviewer B (+1)
- **File**: `firmware/core/OTAUpdater.cpp:549-552`
- **Evidence**: Code explicitly logs "prepareForOTA returned false — continuing anyway" and proceeds to flash firmware, violating AC #2 requirement for hard gate
- **Fix**: Changed failure path to immediately abort, set ERROR state, call `completeOTAAttempt(false)`, and exit task

**Issue 2: `prepareForOTA()` violates Core 0 ownership boundary**
- **Source**: Both reviewers (Reviewer A +3, Reviewer B +3)
- **File**: `firmware/core/ModeRegistry.cpp:483-499`
- **Evidence**: `prepareForOTA()` deletes `_activeMode` on Core 1 while `tick()` on Core 0 may be rendering it. `tick()` does not check `_switchState` before calling `render()` at line 418-420.
- **Fix**: Added explicit `_switchState == SWITCHING` guard in `tick()` before rendering to prevent use-after-free

**Issue 3: Push OTA failures leave `_otaMode` latched, display stuck on "Updating..."**
- **Source**: Reviewer B (+3)
- **File**: `firmware/adapters/WebPortal.cpp:555-713`, failure paths at 692-713
- **Evidence**: After `prepareForOTA()` succeeds at line 656, later failures only abort `Update` and invalidate upload state but never call `completeOTAAttempt(false)` to restore display
- **Fix**: Added `ModeRegistry::completeOTAAttempt(false)` calls to all push OTA failure paths after `prepareForOTA()` succeeds

**Issue 4: `Update.end(true)` failure path skips required `Update.abort()`**
- **Source**: Reviewer B (+1)
- **File**: `firmware/core/OTAUpdater.cpp:806-816`
- **Evidence**: Story AC #8 requires `Update.abort()` on every post-`Update.begin()` failure, but the `Update.end(true)` error path omits it
- **Fix**: Added `Update.abort()` call before error handling in `Update.end(true)` failure path

## Changes Applied

**File**: `firmware/core/OTAUpdater.cpp`
**Change 1**: Hard gate for prepareForOTA() failure (Issue 1)
**Before**:
```cpp
if (!ModeRegistry::prepareForOTA()) {
    LOG_W("OTAUpdater", "prepareForOTA returned false — continuing anyway");
}
```
**After**:
```cpp
if (!ModeRegistry::prepareForOTA()) {
    snprintf(_lastError, ERROR_MAX_LEN, "Could not prepare display for OTA");
    _state = OTAState::ERROR;
    _failurePhase = OTAFailurePhase::DOWNLOAD;
    SystemStatus::set(Subsystem::OTA_PULL, StatusLevel::ERROR, "Display teardown failed");
    ModeRegistry::completeOTAAttempt(false);
    _downloadTaskHandle = nullptr;
    vTaskDelete(NULL);
    return;
}
```
