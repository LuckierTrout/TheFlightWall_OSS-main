- **Claimed Issue**: `updateNmStatus` violates Enforcement Rule #10 by not calling `FW.showToast()` on `!res.body.ok` | **Raised by**: Reviewer A | **Dismissal Reason**: Rule #10 applies to user-initiated actions where silent failure leaves the user without feedback. - **Claimed Issue**: `updateNmStatus` catch handler clears UI state — should preserve last known state on transient errors | **Raised by**: Reviewer D | **Dismissal Reason**: Resetting to "NTP: --" signals a connectivity problem to the user, which is correct and intentional. Preserving stale state would mask actual connectivity issues. The new `setTimeout` schedule in the catch ensures recovery on the next poll.

- **Claimed Issue**: Missing `?_=` cache-buster on `/api/status` poll | **Raised by**: Reviewer A | **Dismissal Reason**: The reboot-polling cache-buster is needed because it fires during a transient network disruption where a cached 200 OK could mask a device that hasn't reconnected. Background status polling under stable conditions does not have this concern. `FW.get()` sends the request to the device's own IP; no intermediary proxy applies.

- **Claimed Issue**: Timezone dropdown underscore→space replacement violates AC #5 ("IANA timezone names as option text") | **Raised by**: Reviewer C | **Dismissal Reason**: AC #5 does not mandate literal underscores. `America/New York` is the standard human-readable display convention used by Google, Apple, and Mozilla for IANA zones. - **Claimed Issue**: `sched_dim_brt` in `collectPayload()` lacks NaN guard | **Raised by**: Reviewers A, D | **Dismissal Reason**: `<input type="range">` enforces `min`/`max` bounds and always produces a numeric value between 0 and 255. `parseInt(rangeEl.value, 10)` on a range input cannot produce NaN under any standard browser behavior. ## Changes Applied

**File**: `firmware/data-src/dashboard.js`
**Change**: `timeToMinutes()` — fix `HH:MM:SS` bug, add range clamp

**Before**:
```js
    if (parts.length !== 2) return 0;
    var h = parseInt(parts[0], 10);
    var m = parseInt(parts[1], 10);
    if (isNaN(h) || isNaN(m)) return 0;
    return h * 60 + m;
```
**After**:
```js
    if (parts.length < 2) return 0;
    var h = parseInt(parts[0], 10);
    var m = parseInt(parts[1], 10);
    if (isNaN(h) || isNaN(m)) return 0;
    var total = h * 60 + m;
    if (total < 0 || total > 1439) return 0;
    return total;
```
