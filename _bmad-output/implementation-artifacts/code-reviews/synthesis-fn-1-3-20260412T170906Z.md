## Issues Dismissed

- **Delayed error response violates AC #2 "validation completes within 1 second"** (Reviewer A, B)
  - **Raised by**: Reviewers A, B
  - **Dismissal Reason**: FALSE POSITIVE for ESPAsyncWebServer architecture. The upload handler returns early on invalid magic byte (line 469), which stops `Update.write()` from being called. However, the HTTP response is deferred until multipart parsing completes — this is a framework constraint, not an implementation bug. The *validation* (magic byte check) completes instantly; only the HTTP *response* is delayed. AC #2's intent is to prevent flash writes on invalid firmware, which is achieved. The 1-second NFR refers to validation logic, not HTTP round-trip time.

- **Use-after-free risk when `clearOTAUpload()` called before `request->send()`** (Reviewer C)
  - **Raised by**: Reviewer C
  - **Dismissal Reason**: FALSE POSITIVE. Code calls `clearOTAUpload()` AFTER `request->send()` completes on line 423, not before. Reviewer misread the sequence. State cleanup happens after response is sent, which is correct.

- **Missing file size validation against partition capacity** (Reviewers A, C)
  - **Raised by**: Reviewers A, C
  - **Dismissal Reason**: FALSE POSITIVE. `Update.begin(partition->size)` (line 484) preallocates the partition capacity, and `Update.write()` enforces bounds internally. If upload exceeds partition size, `Update.end()` will fail with `VERIFY_FAILED`. No additional validation needed — ESP32 Update library handles this. Adding redundant checks would duplicate library logic.

**File**: `firmware/adapters/WebPortal.cpp`

**After**:
```cpp
// Global OTA-in-progress flag (protected by g_otaUploadsMutex)
static bool g_otaInProgress = false;

if (index == 0) {
    // First chunk — validate magic byte and begin update
    {
        std::lock_guard<std::mutex> lock(g_otaUploadsMutex);
        if (g_otaInProgress) {
            // Reject concurrent OTA attempts
            OTAUploadState state;
            state.request = request;
            state.valid = false;
            state.started = false;
            state.bytesWritten = 0;
            state.error = "OTA update already in progress";
            state.errorCode = "OTA_IN_PROGRESS";
            g_otaUploads.push_back(state);
            LOG_W("OTA", "Rejected concurrent upload attempt");
            return;
        }
        g_otaInProgress = true;
    }
    clearOTAUpload(request);
```
