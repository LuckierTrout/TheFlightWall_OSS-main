### 7. Architecture Violation: RenderContext currentFlight Breaks Isolation

- Violates the documented "modes isolated from all subsystems via read-only rendering context"

**Evidence:** Story LE-1.8 added this field and acknowledged coupling in passing ("AC #4: currentFlight must be nullptr when no active flight"). But the architecture decision to add it to RenderContext (vs. a separate FlightDataContext) was not documented as a tradeoff.

### 10. check_size.py Delta Check Logic Uses Naive File I/O Without Existence Guard

**Severity:** MINOR (Error handling)

**File:** `firmware/check_size.py:52`

```python
def get_main_baseline_size(project_dir):
    baseline_path = os.path.join(project_dir, ".firmware_baseline_size")
    if not os.path.exists(baseline_path):
        return None
    try:
        with open(baseline_path, "r") as f:
            return int(f.read().strip())
    except (ValueError, OSError) as e:
        print(f"WARNING: .firmware_baseline_size is unreadable or corrupt ({e}). Delta check skipped.")
        return None
```

The `os.path.exists()` check **races** with the `open()` call (file could be deleted between check and open). Not a practical concern for this artifact, but violates defensive coding patterns.

**Fix:** EAFP pattern (Easier to Ask for Forgiveness than Permission):
```python
try:
    with open(baseline_path, "r") as f:
        return int(f.read().strip())
except FileNotFoundError:
    return None
except (ValueError, OSError) as e:
    print(f"WARNING: .firmware_baseline_size corrupt: {e}")
    return None
```
