## Issues Dismissed

- **Claimed Issue**: Architecture requires a manual POSIX text-input field for unsupported browser timezones | **Raised by**: Reviewer D | **Dismissal Reason**: No AC, task, or architecture document requires a manual override field. AC3 explicitly states "the dropdown defaults to 'UTC' as a safe fallback" — this is the correct specified behavior. Decision F4 describes browser-side IANA→POSIX mapping via `TZ_MAP`, not a freeform POSIX input.

- **Claimed Issue**: Mock server serves `firmware/data-src/` instead of shipped `firmware/data/*.gz` — deployable assets untested | **Raised by**: Reviewer D | **Dismissal Reason**: By design. The mock server is a local developer tool for E2E test iteration against source files. Shipping gz artifacts are validated by the `gzip -9 -c` rebuild step and spot-checked via `gunzip -c` + grep. Requiring the mock server to serve gz files would break the developer workflow.

- **Claimed Issue**: Silent `UTC0` fallback in `saveAndConnect()` (`TZ_MAP[state.timezone] || 'UTC0'`) is a user-invisible failure | **Raised by**: Reviewer B | **Dismissal Reason**: This is the correct defensive last-resort behavior. The upstream bugs (hydration, import) that could corrupt `state.timezone` are now fixed, so this fallback will only fire in genuinely unrecoverable state. The fallback is a safety net, not a bug.

- **Claimed Issue**: `validateStep(3)` should validate timezone against `TZ_MAP` | **Raised by**: Reviewer B | **Dismissal Reason**: The timezone field is a `<select>` element — browser DOM constrains its value to the populated `<option>` values (IANA names from `TZ_MAP`). No free-form input can produce an invalid value without DOM manipulation. Over-engineering for this constrained device wizard UI.

- **Claimed Issue**: DOM reflow storm — 60 `appendChild()` calls without a `DocumentFragment` | **Raised by**: Reviewer B | **Dismissal Reason**: 60 option appends is negligible on any browser built after 2010. The wizard runs once per device setup. No real-world performance regression possible at this scale.

- **Claimed Issue**: Stale comment `// Keys that map to wizard form fields (15 keys)` | **Raised by**: Reviewer A (separately from B/C) | **Dismissal Reason**: NOT dismissed — this was **verified and fixed** (see Medium issues above). Listed here only to note all reviewers saw it.

- **Claimed Issue**: Asset budget risk (headroom < 0.5 KB) | **Raised by**: Reviewer A | **Dismissal Reason**: Informational only. Post-fix total is 49.98 KB (within budget). No code change can address this; if budget is breached in a future story, TZ_MAP pruning is the lever.

## Changes Applied

**File**: `firmware/data-src/wizard.js`
**Change**: Guard `hydrateDefaults()` timezone update against factory default `UTC0`; fix unmapped-IANA fallback

**Before**:
```js
if (d.timezone) {
  state.timezone = POSIX_TO_IANA[d.timezone] || getTimezoneConfig().iana;
  if (wizardTimezone) wizardTimezone.value = state.timezone;
}
```
**After**:
```js
if (d.timezone && d.timezone !== 'UTC0') {
  var tzDet = getTimezoneConfig();
  state.timezone = POSIX_TO_IANA[d.timezone] || (TZ_MAP[tzDet.iana] ? tzDet.iana : 'UTC');
  if (wizardTimezone) wizardTimezone.value = state.timezone;
}
```
