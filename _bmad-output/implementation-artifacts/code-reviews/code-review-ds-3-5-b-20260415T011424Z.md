## 🐛 Correctness & Safety

- **🐛 Bug:** Keyboard focus lost on mode switch error
  - 📍 `firmware/data-src/dashboard.js:1056`
  - 🔄 Reproduction: Focus an idle mode card via keyboard. Press Enter/Space to trigger a mode switch. If the firmware returns an error (or a network timeout occurs), `switchMode()` handles the error by calling `loadDisplayModes()` or `renderModeCards()` to rebuild the DOM. The card element currently holding focus is destroyed in the process, causing keyboard focus to silently reset to the `<body>` element. Users must tab through the entire page from the beginning. Focus choreography is only managed in the success path via `doFocus()`.
