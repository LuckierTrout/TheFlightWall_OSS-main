## 🐛 Correctness & Safety

- **🐛 Race Condition:** `performOtaSelfCheck()` calls `g_wifiManager.getState()` (line 450) from Core 1 (loop task) without synchronization. WiFiManager may update state from WiFi event callbacks (potentially on Core 0 or interrupt context). While `std::atomic` is used for other globals, WiFiManager state access is not synchronized.
  - 📍 `firmware/src/main.cpp:450`
  - 🔄 Reproduction: Difficult to trigger - would require WiFi state change during the brief window of `getState()` execution. More of a theoretical issue given ESP32 WiFi stack behavior, but technically a data race.

- **Potential Overflow:** `millis() - g_bootStartMs` (line 449) can overflow after ~49.7 days. While the 60s timeout makes this practically impossible to hit (boot + 60s << 49 days), defensive coding would use `millis() >= g_bootStartMs + OTA_SELF_CHECK_TIMEOUT_MS` with overflow-aware comparison.
  - 📍 `firmware/src/main.cpp:449`
