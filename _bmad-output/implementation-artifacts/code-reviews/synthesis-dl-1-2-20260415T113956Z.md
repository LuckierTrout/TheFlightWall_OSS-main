## Issues Verified (by severity)

### Critical

- **Issue**: State corruption bug — `onManualSwitch`, `onIdleFallback`, and `onFlightsRestored` update orchestrator `_activeModeId` BEFORE calling `ModeRegistry::requestSwitch()`. If the registry rejects the switch (unknown mode, heap guard failure), orchestrator state diverges from registry truth, violating AC #2's requirement to "keep orchestrator state consistent with registry truth."
  - **Source**: Validator A, Validator B (consensus)
  - **Files**: `firmware/core/ModeOrchestrator.cpp` lines 79-98, 100-115, 117-134
  - **Evidence**: Direct code inspection shows `strncpy(_activeModeId, ...)` before `if (!ModeRegistry::requestSwitch(...))` in all three methods
  - **Fix**: Reordered all three methods to call `requestSwitch()` first, then update state fields only on success. Added early return on failure to prevent state update.
