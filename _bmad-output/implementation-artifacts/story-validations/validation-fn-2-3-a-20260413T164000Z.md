### 3. Add Error Handling for Status Polling

**Impact:** Poor user experience if polling fails silently
**Source:** Task 6

**Current Gap:**
The `updateNmStatus()` function has a `.catch()` block that only clears the status text but doesn't inform the user. If the device goes offline or `/api/status` becomes unavailable, users won't know why status stopped updating.

**Recommended Fix:**
Add error handling that shows a toast message on first poll failure (not on every failure to avoid spam):
```js
var firstPollFailed = false;
function updateNmStatus() {
  FW.get('/api/status').then(function(res) {
    // ... existing success handling ...
  }).catch(function() {
    if (!firstPollFailed) {
      FW.showToast('Cannot reach device to update status', 'warning');
      firstPollFailed = true;
    }
    // ... existing status clearing ...
  });
}
```

### 4. Add Accessibility Requirements for Status Indicators

**Impact:** WCAG compliance and screen reader support
**Source:** AC #6, #7

**Current Gap:**
The NTP badge and schedule status indicators lack ARIA attributes. Screen readers won't announce status changes to users. This is particularly important for "Currently dimming" state changes.

**Recommended Fix:**
Add ARIA attributes to status elements:
```html
<span class="nm-ntp-badge" id="nm-ntp-badge" aria-live="polite">NTP: --</span>
<span class="nm-sched-status" id="nm-sched-status" aria-live="polite"></span>
```

Also add `aria-label` to the checkbox for better accessibility:
```html
<label class="nm-toggle-label">
  <input type="checkbox" id="nm-enabled" aria-label="Enable Night Mode schedule">
  <span>Enable Schedule</span>
</label>
```
