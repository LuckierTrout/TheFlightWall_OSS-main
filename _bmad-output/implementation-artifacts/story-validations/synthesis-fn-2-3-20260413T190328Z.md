## Changes Applied

**Location**: `_bmad-output/implementation-artifacts/stories/fn-2-3-dashboard-night-mode-card.md` — Task 6.1, schedule status block

**Change**: Added `!s.ntp_synced` as the first condition in the schedule status branch so that the explanatory note required by AC #6 is shown, and corrected the `schedule_active` branch text to match AC #7 exactly.

**Before**:
```js
// Schedule status
if (s.schedule_dimming) {
  nmSchedStatus.textContent = 'Currently dimming';
  ...
} else if (s.schedule_active) {
  nmSchedStatus.textContent = 'Schedule active';
  ...
```
**After**:
```js
// Schedule status (NTP must be synced for schedule to run)
if (!s.ntp_synced) {
  nmSchedStatus.textContent = 'Schedule paused — waiting for NTP sync';
  nmSchedStatus.className = 'nm-sched-status nm-inactive';
} else if (s.schedule_dimming) {
  nmSchedStatus.textContent = 'Currently dimming';
  ...
} else if (s.schedule_active) {
  nmSchedStatus.textContent = 'Schedule active (not in dim window)';
  ...
```
